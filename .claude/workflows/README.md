# ECMABasic Workflows

Workflows are **multi-agent orchestration scripts** that coordinate complex tasks across multiple specialized agents.

## What Are Workflows?

Workflows are JavaScript files that:
- Spawn multiple agents in parallel or sequence
- Coordinate data flow between agents
- Aggregate results from multiple sources
- Handle complex, multi-step automation

## When to Use Workflows

⚠️ **Workflows are heavy** - they spawn many agents and consume significant tokens. Only use when:

1. **User explicitly requests orchestration**:
   - "Use a workflow to..."
   - "Fan out agents to..."
   - "Run multi-agent analysis..."

2. **Task genuinely requires parallelism**:
   - Review 20+ files independently
   - Test multiple hypotheses simultaneously
   - Aggregate data from many sources

3. **Single agent can't fit context**:
   - Codebase too large for one agent
   - Need independent perspectives
   - Adversarial verification needed

## When NOT to Use Workflows

❌ Don't use workflows for:
- Simple, linear tasks (use skills instead)
- Tasks a single agent can handle
- When user hasn't opted in to multi-agent work
- Exploratory work (scout first, orchestrate later)

## How Workflows Are Created

Workflows are **generated by Claude**, not hand-written:

1. User requests multi-agent work
2. Claude writes workflow script
3. Script is saved to this directory
4. Workflow executes with progress tracking
5. Results aggregated and reported

## Workflow Files

Generated workflows appear as:
- `workflow-{timestamp}.js` - One-time workflows
- `saved-workflow-name.js` - Named, reusable workflows

## Triggering Workflows

### Option 1: Explicit Request
```
User: "Use a workflow to review all 128 C# files for nullable violations"
```

### Option 2: Ultracode Mode
Enable in settings for automatic workflow usage on complex tasks.

### Option 3: Saved Workflow
```
User: "Run the modernization workflow"
```

## Budget Awareness

Workflows respect token budgets:
```javascript
// User types: "Review all files +500k"
// Workflow scales to stay within 500k token budget
while (budget.remaining() > 50_000) {
  // Spawn agents dynamically
}
```
